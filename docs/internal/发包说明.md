# SRW DeepSeek 本地桌面网关发包说明

## 1. 目标

本说明用于指导管理员或发布人员将项目打成可直接在 Windows 上运行的 GUI 程序，而不是把源码分发到每台电脑。

## 2. 发布产物

执行打包脚本后，会生成以下目录：

- `dist/release/DeepSeekDesktopGateway.exe`：单文件程序，可直接复制到其他电脑运行
- `dist/release/*.md`：配套使用/部署/运维文档

最终可交付给终端用户的是包含 `DeepSeekDesktopGateway.exe` 的整个发布目录；程序的默认数据目录就是 EXE 所在目录。

## 3. 打包步骤

### 3.1 执行 Python 一键打包脚本

推荐直接执行以下命令：

```powershell
Set-Location d:\github_workstation\litellm
py .\scripts\build_release.py
```

如果当前机器的 `py` 命令不可用，也可以使用任意可用 Python 解释器执行该脚本。

### 3.2 备用 PowerShell 打包脚本

```powershell
Set-Location d:\github_workstation\litellm
PowerShell -ExecutionPolicy Bypass -File .\scripts\build_release.ps1
```

### 3.3 打包脚本会完成的事情

1. 创建或复用 `.venv`
2. 安装运行依赖
3. 从本地 `vendor/litellm` 安装 LiteLLM
4. 安装 `pyinstaller`
5. 依据 `packaging/pyinstaller.spec` 生成单文件 `DeepSeekDesktopGateway.exe`
6. 把部署、使用、运维文档复制到发布目录
7. 执行发布前自检，确认 EXE 可以成功拉起

### 3.4 发布前自检会检查什么

1. PyInstaller 分析结果中必须包含 `tiktoken_ext.openai_public`
2. `DeepSeekDesktopGateway.exe` 必须能够被实际拉起，并在 5 秒后仍保持运行

如果以上任一检查失败，打包脚本会直接报错退出，避免继续分发“能打包但不能启动”的发布包。

## 4. 终端用户如何启动

终端用户无需源码环境，只需要：

1. 将整个发布目录复制到任意目录
2. 双击 `DeepSeekDesktopGateway.exe` 即可

## 5. 建议的交付方式

推荐直接复制整个发布目录。这样配置文件、数据库、日志和报表都能跟随迁移；但上游 API Key 绑定当前 Windows 用户，换电脑或换账户后需要重新录入。

## 6. 数据保存位置

发包运行后，程序默认会把运行数据直接写到 EXE 所在目录。

其中包含：

- `config.json`
- `usage.db`
- `secrets.json`（绑定当前 Windows 用户的密钥载荷）
- `reports` 目录
- `logs` 目录
- `runtime` 目录

应用的“运维”页还提供：

- 日志查看
- 数据备份
- 数据恢复

这样做的目的：

1. 避免安装目录因权限问题不可写
2. 便于直接复制整个文件夹做迁移或发放模板
3. 便于单独备份数据库与历史报表

如需改到其他位置，可在启动前设置环境变量 `SRW_GATEWAY_DATA_DIR`。

## 7. 验收要求

打包完成后，建议至少验证：

1. `DeepSeekDesktopGateway.exe` 可以直接双击打开 GUI
2. 数据能写入 EXE 所在目录
3. VS Code 插件可以通过本地网关正常发起请求
