# 架构说明

## 1. 总体目标

本项目用于在每台 Windows 笔记本上部署一个以 LiteLLM Proxy 为内核的本地桌面网关，统一承接 VS Code 中 DeepSeek Copilot Chat 的请求，并在本机完成费用统计、预算限制、管理员保护和报表导出。

## 2. 总体架构

系统由四层组成：

1. VS Code 插件接入层：负责把 DeepSeek Copilot Chat 请求发送到本机网关。
2. LiteLLM 内核层：负责 OpenAI 兼容协议、模型路由、调用上游和请求生命周期回调。
3. 桌面管理界面层：负责设置、状态展示、自启动和报表导出。
4. 本地数据与凭据层：负责保存本地摘要日志、配置文件和密钥。

## 3. 核心模块

### 3.1 图形界面层

文件：`src/deepseek_gateway/gui.py`

职责：

- 展示网关状态
- 修改预算和设备信息
- 保存上游地址与密钥
- 导出 CSV 和 XLSX 报表
- 控制网关启动与停止

### 3.2 LiteLLM 内核承载层

文件：`src/deepseek_gateway/gateway_service.py`

职责：

- 动态生成 LiteLLM Proxy 运行配置
- 托管 LiteLLM data-plane 路由并统一对外提供 OpenAI 兼容接口
- 通过 LiteLLM callback 承载预算预检查和本地摘要记账
- 复用 LiteLLM 的 OpenAI 兼容协议、模型路由和上游调用能力

### 3.3 配置管理层

文件：`src/deepseek_gateway/config_manager.py`

职责：

- 保存本地配置
- 为配置文件生成和校验完整性签名
- 保存程序同目录的 `secrets.json`
- 将上游密钥和签名密钥以绑定当前 Windows 用户的受保护载荷写入 `secrets.json`

### 3.4 安全层

文件：`src/deepseek_gateway/security.py`

职责：

- 生成管理员密码哈希
- 校验管理员密码

### 3.5 数据层

文件：`src/deepseek_gateway/database.py`

职责：

- 保存每次请求明细
- 计算当月累计费用
- 导出月度 CSV 和 XLSX 报表

### 3.6 本地 LiteLLM 源码层

目录：`vendor/litellm`

职责：

- 提供 Proxy/Gateway 内核能力
- 提供官方成本计算、回调、analytics 和 spend logs 能力
- 作为本地固定源码依赖存在，避免只依赖远程安装

## 4. 数据流

一次正常请求的数据流如下：

1. VS Code 插件把请求发送到本地 `http://127.0.0.1:8765/v1/chat/completions`。
2. 桌面层生成的 LiteLLM 运行配置决定模型映射、上游地址和回调策略。
3. LiteLLM 在 pre-call callback 中执行本机预算检查。
4. LiteLLM 使用本地保存的上游 API Key 调用 DeepSeek。
5. LiteLLM 在 success/failure callback 中把本地摘要写入 `usage.db`。
6. 结果由 LiteLLM 直接返回给 VS Code。

## 5. 安全设计

### 5.1 配置保护

- 打包后的 EXE 模式默认把配置文件保存到 EXE 所在目录的 `config.json`
- 文件内容使用 HMAC 做完整性签名
- 如果终端用户直接修改配置文件，将触发校验失败

### 5.2 密钥保护

- 上游 API Key 不写入配置文件
- 上游 API Key 存入程序同目录的 `secrets.json`
- `secrets.json` 中的敏感字段不以明文 JSON 值保存，并绑定当前 Windows 用户
- 如果目录复制到另一台电脑，或改由其他 Windows 账户运行，需要重新录入上游 API Key

### 5.3 管理员保护

- 设置敏感项前要求输入管理员密码
- 管理员密码以哈希形式保存，不保存明文
