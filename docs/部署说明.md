# SRW DeepSeek 本地桌面网关部署说明

## 1. 运行环境要求

- 操作系统：Windows 10 或 Windows 11（64 位）
- 网络：本机可访问 DeepSeek 上游接口
- 权限：允许写入程序所在目录、允许设置开机自启动（注册表）

## 2. 部署方式

### 2.1 首次部署

1. 将发布包（包含 `DeepSeekDesktopGateway.exe` 及配套文档）复制到目标电脑的任意目录。
2. 双击 `DeepSeekDesktopGateway.exe` 启动程序。
3. 程序会自动在 EXE 所在目录创建运行所需的数据文件（配置文件、数据库、日志等）。

### 2.2 迁移到新电脑

1. 将整个程序目录（含 EXE 及所有数据文件）复制到新电脑的任意位置。
2. 双击 `DeepSeekDesktopGateway.exe` 启动。
3. 程序会保留原有的配置和日志数据，但上游 API Key 绑定原 Windows 用户，新电脑需要重新录入。

### 2.3 自定义数据目录

如需将数据库与配置保存到其他位置，可在启动前设置环境变量 `SRW_GATEWAY_DATA_DIR`，指向目标目录。

## 3. 首次初始化流程

程序首次启动后，需要由管理员完成以下操作：

1. 软件启动后不会弹出管理员密码提示框。
2. 由发布管理员掌握管理员密码后，进入"设置"页。
3. 填写或确认设备名称与设备编号。
4. 配置本机监听地址和端口。
5. 设置月度预算金额（人民币）。
6. 填写上游 DeepSeek 基础地址。
7. 填写上游 DeepSeek API Key。
8. 按需要调整各模型缓存命中输入价、普通输入价和输出价，单位为人民币/百万 tokens。
9. 根据需要勾选开机自启动。
10. 点击"保存设置"，返回"概览"页，点击"启动网关"。

## 4. VS Code 侧接入方式

在 VS Code 的 `settings.json` 中加入以下配置：

```json
{
  "deepseek-copilot.baseUrl": "http://127.0.0.1:8765/v1",
  "deepseek-copilot.modelIdOverrides": {
    "deepseek-v4-flash": "deepseek-v4-flash",
    "deepseek-v4-pro": "deepseek-v4-pro"
  }
}
```

如需更换端口，请同步修改 `baseUrl`。

## 5. 数据落盘位置

程序默认会将运行数据保存在 EXE 所在目录下：

| 文件 / 目录 | 说明 |
|---|---|
| `config.json` | 本地配置文件（带完整性签名） |
| `usage.db` | 请求日志数据库（SQLite） |
| `secrets.json` | 上游 API Key 与签名密钥（绑定当前 Windows 用户） |
| `reports/` | 导出的月报文件目录 |
| `logs/` | 运行日志目录 |

> 注意：`secrets.json` 中的敏感字段通过 Windows 系统级加密保护，绑定当前登录用户。将目录复制到其他电脑或由其他账户运行时，需要重新录入上游 API Key。

## 6. 运维能力

程序提供以下运维功能（位于 GUI 的"运维"页）：

- 查看最新运行日志
- 打开数据目录和日志目录
- ZIP 备份与恢复
- 管理员密码保护的数据重置

## 7. 升级流程

建议按以下步骤升级：

1. 在"运维"页备份数据，或直接备份整个程序目录。
2. 用新版 `DeepSeekDesktopGateway.exe` 替换旧版。
3. 启动新版程序并验证预算、报表和 VS Code 接入是否正常。
