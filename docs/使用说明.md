# SRW DeepSeek 本地桌面网关使用说明

## 1. 软件用途

本软件用于在每台研发笔记本电脑上建立以 LiteLLM 为内核的本地模型网关，承接 VS Code DeepSeek Copilot Chat 的调用请求，并实现：

- 本机费用统计
- 月度费用上限控制
- 本地报表导出
- 历史月度消费查看
- 日志查看与数据备份恢复
- 管理员保护配置

## 2. 主界面说明

主界面包含五个标签页：

### 2.1 概览

概览页展示：

- 网关运行状态
- 当前设备名称和设备编号
- 本地监听地址
- 当月累计费用与预算上限

概览页支持：

- 启动网关
- 停止网关
- 刷新状态

### 2.2 设置

设置页用于维护以下内容：

- 设备名称
- 设备编号
- 监听地址
- 监听端口
- 月度预算金额（人民币）
- 上游基础地址
- 上游 API Key
- 开机自启动
- 管理员密码
- 各模型缓存命中输入单价（人民币/百万 tokens）
- 各模型输入/输出单价（人民币/百万 tokens）

说明：

1. 打开“设置”页前必须先输入管理员密码。
2. 管理员密码不会在启动界面或首次打开时弹窗显示，应由发布管理员单独保管。
3. 如需修改默认管理员密码，请调整 [src/deepseek_gateway/security.py](src/deepseek_gateway/security.py) 中的 `DEFAULT_ADMIN_PASSWORD` 常量并重新发包。
4. 如果已经设置本地管理员密码，保存设置前可输入当前管理员密码；若当前输入框为空，会复用本次会话已经验证过的密码。
5. 新管理员密码只在输入了“新管理员密码”时才会更新。
6. 上游 API Key 不会显示历史值，输入新值后会覆盖原值。
7. 当前桌面版会把缓存命中输入价、普通输入价和输出价同时写入 LiteLLM runtime config，并使用 LiteLLM 返回的 `response_cost` 做本地预算与报表记账。

### 2.3 报表

报表页支持导出：

- 任选月份 CSV 明细报表
- 任选月份 XLSX 明细报表

导出的报表可直接交给助理做月度统计。

### 2.4 历史

历史页用于查看每个月的累计消费记录，默认展示：

- 月份
- 累计消费金额（人民币）
- 成功请求次数
- 失败请求次数

### 2.5 运维

运维页提供以下能力：

- 查看最新日志内容
- 打开数据目录
- 打开日志目录
- 备份数据为 ZIP 文件
- 从 ZIP 文件恢复数据
- 输入管理员密码后重置本地数据库

## 3. 常规使用流程

建议终端用户按以下顺序使用：

1. 启动本软件。
2. 在概览页确认网关状态为“运行中”。
3. 打开 VS Code 并使用 DeepSeek Copilot Chat。
4. 月末由管理员或助理在报表页选择月份并导出报表。
5. 需要追踪趋势时，可在“历史”页查看历月累计消费记录。
6. 需要备份或恢复数据时，可在“运维”页执行相应操作。

## 4. 预算控制逻辑

当前版本按“设备月度预算”进行限制：

1. 每次请求进入 LiteLLM 内核前，系统先读取当前月累计费用。
2. 如果累计费用已达到或超过预算上限，请求会在 LiteLLM pre-call hook 阶段被阻断。
3. 成功或失败请求会通过 LiteLLM callback 写入本地 SQLite 摘要数据库。

## 5. 报表字段说明

导出的明细报表包含以下字段：

- 时间戳
- 设备编号
- 网关模型名
- 上游模型名
- 输入 Token
- 输出 Token
- 费用（人民币）
- 状态
- 错误信息

## 6. 常见问题

### 6.1 网关已启动但 VS Code 无法访问

请检查：

1. `settings.json` 中的 `baseUrl` 是否和本地监听地址一致。
2. 本机端口是否被其他程序占用。
3. 本地防火墙是否阻断本机回环访问。

### 6.2 保存设置时报管理员密码错误

说明当前输入的管理员密码与本机保存的密码不一致，需要使用正确密码后再保存。

### 6.3 报表导出后金额为 0

请检查：

1. 本月是否真的产生过成功请求。
2. 请求是否因预算超额、上游异常或 LiteLLM 回调初始化失败而失败。
3. 本地数据库是否可正常写入。

### 6.4 本地数据库保存在哪里

打包后的 EXE 默认会把运行数据保存在 EXE 所在目录下：

- `DeepSeekDesktopGateway.exe` 所在目录

其中主要文件包括：

- `config.json`
- `usage.db`
- `secrets.json`
- `reports` 目录
- `logs` 目录
- `runtime` 目录

说明：

1. 直接复制包含 EXE 的整个文件夹到新电脑，可一并迁移配置、数据库、日志、报表和本地保存的上游密钥。
2. 如果只复制 `DeepSeekDesktopGateway.exe` 或只复制 `usage.db`，迁移不会完整，因为 `config.json` 和 `secrets.json` 也在同目录。
3. 如需改到其他磁盘，可预先设置环境变量 `SRW_GATEWAY_DATA_DIR`。
4. `secrets.json` 中的敏感字段不会以明文 JSON 值保存，并绑定当前 Windows 用户。
5. 如果目录复制到另一台电脑，或改由其他 Windows 账户运行，程序需要重新录入上游 API Key。
6. 源码开发模式仍默认使用用户文档目录，不受上述 EXE 便携模式影响。

### 6.5 如何备份和恢复数据

1. 打开“运维”页。
2. 点击“备份数据”，保存生成的 ZIP 文件。
3. 如需恢复，点击“恢复数据”，选择之前的备份 ZIP。
4. 恢复完成后建议关闭并重新启动程序。

### 6.6 如何保留配置并清空历史调用记录

1. 打开“运维”页。
2. 点击“重置数据库”。
3. 输入管理员密码。
4. 确认后程序会清空 `usage.db` 中的请求记录，但保留 `config.json`、`secrets.json`、日志和报表文件。
